# Open Questions

## 1. 기술 결정 필요
- 상태 저장 확장
  - 후보: 서버 메모리 단독 vs Redis 추가
  - 기준: 복구/확장성

## 2. 제품 결정 필요
- 초기 동시 에이전트 목표 수
- 오피스 스타일(픽셀/SVG/일러스트) 우선순위
- 경고 UX 방식(색상 강조 vs 토스트 vs 알림 패널)

## 3. 데이터 입력원 확정
- Hooks만 사용할지,
- SDK 스트리밍까지 병행할지

## 4. 보안/운영
- 내부 인증 방식(토큰/사내 SSO)
- 로그 보존 기간
- 민감 데이터 마스킹 정책

## 5. 리스크
- 입력 이벤트 다양성으로 인한 스키마 변동
- 실시간 렌더링 성능 저하
- 상태 불일치(서버 상태 vs 클라이언트 로컬 애니메이션)

## 6. 완화 전략
- 스키마 버전 + 변환기 도입
- 성능 예산 기반 최적화 순서 고정
- snapshot 주기 동기화 및 강제 리셋 기능 제공
