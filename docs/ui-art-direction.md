# UI Art Direction (Docs Only)

현재 문서는 디자인 구현이 아니라 방향 고정 목적이다.

## 1. 목표 톤
- "오피스 운영 게임" 분위기
- 조작형 게임이 아니라 운영 관측형 UI
- 정보 UI와 배경 아트가 충돌하지 않도록 레이어 분리

## 2. 오피스 탭 방향
- 배경:
  - 타일/카펫/벽면 톤을 가진 실내 배경
  - 탕비실/업무구역/동선 구분이 색감으로 명확해야 함
- 캐릭터:
  - 런타임 조합형 아바타(헤어/상의/액세서리)
  - 팀장은 고정 마커(예: 왕관 배지, 금색 아웃라인)
- 소품:
  - 책상, 모니터, 화분, 프린터, 탕비실 기기
  - 상태 이펙트(working/failed/resting)는 소품과 겹쳐도 읽혀야 함

## 3. 대시보드 탭 방향
- 게임풍보다 "운영 콘솔" 우선
- 원칙:
  - 카드 1차 요약, 상세는 패널 드릴다운
  - 색상 의미 고정(정상/경고/실패)
  - 타임라인 가독성 우선(행간/그룹/필터)

## 4. 에이전트 탭 방향
- "이력서/프로필" 톤
- 구성:
  - 좌측: 목록/필터
  - 우측: 프로필 헤더(사진/직책/배지), 스킬, 최근 작업 이력
- 계약직/정직원 라벨은 시각적으로 명확히 분리

## 5. 설정 탭 방향
- 도구 설정 패널 느낌
- 훅 온보딩, 언어/모션, 연동 상태를 카드 섹션으로 분리
- 위험 작업(자동 설정 시도)은 강조색 구분

## 6. 구현 전 결정 필요 항목
- 아트 스타일:
  - 픽셀아트 / 벡터 일러스트 / 2.5D 중 선택
- 타이포:
  - 본문/제목 폰트 쌍 확정
- 캐릭터 조합 규칙:
  - 랜덤 시드 방식(에이전트 ID 기반 고정 생성)
- 아이콘 팩:
  - 상태/도구/경고 아이콘 세트 확정
