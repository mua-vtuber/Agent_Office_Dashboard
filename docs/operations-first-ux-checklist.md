# Operations-First UX Checklist

목적: 연출을 유지하되, 운영 도구로서의 가치를 우선 보장한다.

## 1) 정보 우선성
- [ ] 첫 화면 3초 내 `누가 / 무엇을 / 왜 멈췄는지` 파악 가능
- [ ] 실패/지연 경고가 장식 요소보다 먼저 인지됨
- [ ] 에이전트 상세 진입 없이도 핵심 상태 요약 확인 가능

## 3) 생산성 영향
- [ ] 실패 원인 탐색 클릭 수가 기존 대비 감소
- [ ] Time Travel + 세션 필터로 분석 시간 단축 확인
- [ ] 운영자가 다음 액션(재시도/할당변경/대기판단)을 1분 내 결정 가능

## 5) 성능/신뢰성
- [ ] 연출 on/off와 무관하게 핵심 상태/이벤트 표시 동일
- [ ] 이벤트 반영 지연이 `performance-targets.md` 목표 충족
- [ ] 이벤트 유실/중복 처리 규칙이 실제 로그에서 검증됨
- [ ] 재연결 후 snapshot 동기화로 상태 불일치 자동 복구

## 적용 규칙
- 신규 UI/모션 추가 시 이 문서 체크 항목을 먼저 통과해야 반영한다.
- 체크리스트 결과는 PR/작업 로그에 간단히 남긴다.
